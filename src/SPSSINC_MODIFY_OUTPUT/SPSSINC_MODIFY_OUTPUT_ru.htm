Изменить заголовки вывода

Используйте данный диалог, чтобы изменить заголовки и/или текст на панели схемы в Viewer, а также заголовки или текст самих элементов. Вы также можете изменить их видимость. Вы задаете типы элементов для обработки и критерии отбора в главном диалоговом окне, а также выполняемые действия во вспомогательном диалоговом окне Действия. Эти операции применяются к назначенному окну Viewer.

Выберите один или более типов элементов для обработки. Большинство из них говорят сами за себя. Заголовки являются элементами схемы, которые имеют дочерние элементы, расположенные ниже них. Title-заголовки (или просто заголовки) являются заголовками реальных элементов. Не все элементы имеют заголовки, которые можно изменить данной командой. В частности, заголовки элементов Диаграммы, Модели и Деревья не могут быть изменены. Однако их текст в схеме изменить можно.

По умолчанию команда обрабатывает вывод из первой предшествующей процедуры (игнорируя внедряющиеся элементы журнала). Это можно изменить, чтобы обработать все текущее содержимое Viewer.

Обрабатываются только видимые элементы. Можно скрыть выбранные элементы, выбирая Скрыть во вспомогательном диалоговом окне Действия.

Критерии выбора

Можно ограничить обрабатываемые элементы, задавая одно или более из Команда, Подтип таблицы, Критерии заголовков схемы и Критерии заголовков элементов. Поля, которые не применяются к конкретному типу объектов, игнорируются при отборе. Модифицируются только те элементы, которые удовлетворяют всем применимым критериям.

Поля Команда и Подтип таблицы соответствуют OMS-идентификаторам для элементов. Их можно найти, щелкая правой кнопкой мыши по записи схемы примера или в Утилиты>OMS-идентификаторы.

Критерии заголовков схемы и Критерии текста элементов можно задать как точные совпадения (игнорируя регистр), а также с помощью текста начала или конца. Можно также использовать регулярное выражение, что является обычным способом описания паттернов. Регулярные выражения здесь не объясняются. Полезным источником информации является http://www.amk.ca/python/howto/regex.

Вспомогательный диалог Действия

Щелкните по кнопке Действия, чтобы задать, что делать с выбранными элементами. Если по Действия ничего не задано, то команда не выполнит никаких действий, не предупреждая об этом.

Замена текста

Можно ввести новый текст для заголовка схемы и/или заголовка элемента, или текст в группах Текст схемы и Текст элемента. Это может быть буквенный текст для замены старого содержимого, или это может быть регулярное выражение замены.

В новом тектсе можно ссылаться на старый текст, а также можно включить порядковый номер или букву. Порядковый номер или буква представляется с помощью "\\0", а старый текст с помощью "\\1". Например, чтобы добавить порядковый номер к старому заголовку, можно ввести
Таблица \\0: \\1
Обозначение \\0 можно также использовать в регулярных выражениях.

В тексте замены можно использовать обозначение форматирования html или rtf. Например,
<html>The <i>New</i> Title
заставит вывести "New" курсивом.

В критериях отбора информация о форматировании игнорируется. Директива \\1 также пропускает информацию о форматировании.

При использования круглых скобок в регулярном выражении отбора можно использовать ссылки \nна эти группы, где n является номером группы. Если для отбора не использовалось регулярное выражение, старый заголовок рассматривается как группа 1.

Последовательная нумерация

Значением последовательности может быть число, буква или римская цифра. Поставьте переключатель Цифровые или буквенные элементы для выбора. По умолчанию стиль является числовым и начальным значением является 1. Можно ввести другое начальное значение. Если это одна или несколько букв (до четырех), то последовательность будет буквами в регистре, задаваемом первой буквой. Если выбрать одну из кнопок римских цифр, то последовательность будет состоять из римских цифр. В этом случае начальное значение, если оно введено, должно быть числовым.

Чтобы получить значения последовательности, нужно поставить флажок Цифровые или буквенные элементы, а также включить \\0 в текст схемы или элемента.

Вставка разрывов страниц

Если выбрать Заголовки в группе Объекты вывода в главном диалоговом окне, то можно вставить разрыв страницы впереди каждого заголовка, который удовлетворяет критериям отбора. Поставьте переключатель Разрывы страниц. По умолчанию действию подвергаются только заголовки на уровне схемы с отступом на один уровень под заголовком (heading) процедуры. Выберите кнопку Все, чтобы установить разрывы для всех объектов заголовков на всех уровнях схемы.

Дополнительные функции

Программисты на языке Python могут задать дополнительную функцию, чтобы она вызывалась для каждого выбранного объекта. Это может уберечь от написания кода для перехода к соотвествующим элементам. Вставьте применяемую функцию в виде
имя модуля.имя функции

Дополнительные функции могут иметь параметры. Чтобы задать один или несколько параметров, заключите список параметров в круглые скобки, и напишите имена и значения параметров. Например,
myfunctions.writeitemtitles(file='filespec')
вызовет функцию writeitemtitles в модуле myfunctions. Обратитесь к документации по любой конкретной дополнительной функции, чтобы узнать, какие параметры она принимает. Имена параметров должны быть написаны в регистре, указанном в определении функции.

Если значение параметра является строкой, заключите ее в одиночные кавычки, например,
option='both'

Дополнительные возможности

Этот диалог формирует синтаксис для команды расширения SPSSINC MODIFY TABLES. Чтобы вывести справку для этой команды, введите следующую команду:

SPSSINC MODIFY TABLES /HELP.

Этот командный синтаксис позволяет:
  • Задать список команд для отбора
  • Задать список подтипов таблиц
  • Обрабатывать видимые, невидимые или все элементы вывода
  • Удалять выбранные элементы, вместо того чтобы их скрывать
