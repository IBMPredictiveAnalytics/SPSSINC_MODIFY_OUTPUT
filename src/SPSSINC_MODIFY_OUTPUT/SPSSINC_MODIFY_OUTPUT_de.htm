Ausgabetitel ändern

In diesem Dialog können Sie die Titel und/oder den Text im Viewer-Gliederungsfenster sowie die Titel oder den Text der Elemente selbst ändern. Sie können auch ihre Sichtbarkeit ändern. Im Hauptdialog legen Sie die zu verarbeitenden Objekttypen und die Auswahlkriterien fest, im Unterdialog Aktionen die anzuwendenden Aktionen. Die Operationen werden auf das entsprechende Viewer-Fenster angewendet.

Wählen Sie einen oder mehrere Elementtypen zur Bearbeitung aus. Die meisten sind offensichtlich. Überschriften sind die Gliederungselemente, unter denen sich Unterpunkte befinden. Die Titel sind die Titel der eigentlichen Elemente. Nicht alle Elemente haben Titel, die mit diesem Befehl geändert werden können. Insbesondere die Titel von Diagrammen, Modellen und Bäumen können nicht geändert werden. Beim Gliederungstext ist dies aber möglich.

Standardmäßig verarbeitet der Befehl die Ausgabe der unmittelbar vorangegangenen Prozedur (und ignoriert dazwischen liegende Protokollelemente). Sie können dies ändern, um den gesamten aktuellen Inhalt des Viewers zu verarbeiten.

Nur sichtbare Elemente werden verarbeitet. Sie können die ausgewählten Elemente ausblenden, indem Sie Ausblenden im Unterdialog 'Aktionen' wählen.

Auswahlkriterien

Sie können die verarbeiteten Elemente einschränken, indem Sie eine oder mehrere der Optionen 'Befehl', 'Tabellenuntertyp', 'Gliederungstitelkriterien' und 'Elementtitelkriterien' angeben. Felder, die nicht auf einen bestimmten Objekttyp zutreffen, werden beim Bestimmen der Auswahl ignoriert. Nur Elemente, die alle anwendbaren Kriterien erfüllen, werden geändert.

Die Felder Befehl und Tabellenuntertyp entsprechen den OMS-IDs für die Elemente. Sie finden diese, indem Sie mit der rechten Maustaste auf den Gliederungseintrag eines Beispiels klicken oder über Dienstprogramme>OMS-IDs.

Die Gliederungstitelkriterien und Elementtitelkriterien können als exakte Übereinstimmungen (ohne Berücksichtigung der Groß- und Kleinschreibung) oder durch den Anfangs- oder Endtext angegeben werden. Sie können auch einen regulären Ausdruck verwenden, der eine allgemeine Art der Beschreibung eines Musters ist. Reguläre Ausdrücke werden hier nicht erklärt. Über folgenden Link erhalten Sie weitere Informationen: http://www.amk.ca/python/howto/regex.

Unterdialog 'Aktionen'

Klicken Sie auf die Schaltfläche Aktionen, um festzulegen, was mit den ausgewählten Elementen geschehen soll. Wenn unter 'Aktionen' nichts angegeben ist, wird der Befehl nicht aktiv.

Ersetzen von Text

Sie können einen neuen Text für die Gliederung und/oder den Elementtitel oder Text in den Gruppen Gliederungstext und Elementtext eingeben. Dabei kann es sich um Literaltext handeln, der den alten Inhalt ersetzt, oder um einen regulären Ausdruck für die Ersetzung.

Im neuen Text können Sie auf den alten Text verweisen und eine laufende Nummer oder einen Buchstaben einfügen. Die laufende Nummer oder der Buchstabe wird durch "\\0" und der alte Text durch "\\1" dargestellt. Um zum Beispiel eine laufende Nummer zum alten Titel hinzuzufügen, könnten Sie Folgendes eingeben:
Tabelle \\0: \\1
Sie können die Notation \\0 auch in einem regulären Ausdruck verwenden.

Sie können im Ersetzungstext das HTML- oder RTF-Format verwenden. Beispiel:
<html>Der <i>neue</i> Titel
bewirkt, dass "neue" kursiv dargestellt wird.

Bei den Auswahlkriterien werden die Formatierungsinformationen ignoriert. Die Anweisung \\1 übergeht die Formatinformationen ebenfalls.

Wenn Sie im regulären Ausdruck der Auswahl Klammern verwendet haben, können\n Sie Verweise auf diese Gruppen verwenden, wobei n die Gruppennummer ist. Wurde kein regulärer Ausdruck für die Auswahl verwendet, gilt der alte Titel als Gruppe 1.

Sequenznummer

Der Sequenzwert kann eine Zahl, Buchstaben oder römische Ziffern sein. Die Auswahl können Sie über Zahlen- oder Buchstabenelemente vornehmen. Standardmäßig sind die Stile numerisch und beginnen bei 1. Sie können einen anderen Startwert eingeben. Handelt es sich um einen oder mehrere Buchstaben (bis zu vier), richten sich alle Buchstaben hinsichtlich der Groß- und Kleinschreibung nach dem ersten Buchstaben. Wenn Sie eine der Schaltflächen mit den römischen Ziffern auswählen, wird die Folge in römischen Ziffern angegeben. Der Startwert, sofern eingegeben, muss in diesem Fall numerisch sein.

Um Sequenzwerte zu erhalten, müssen Sie das Kästchen Zahlen- oder Buchstabenelemente auswählen und \\0 in den Gliederungs- oder Positionstext einfügen.

Einfügen von Seitenumbrüchen

Wenn Sie Titel in der Gruppe Ausgabeobjekte im Hauptdialog auswählen, können Sie vor jedem Titel, der den Auswahlkriterien entspricht, einen Seitenumbruch einfügen. Wählen Sie das Feld Seitenumbrüche aus. Standardmäßig sind nur Titel auf der Gliederungsebene betroffen, die eine Ebene unter einer Prozedurüberschrift eingerückt sind. Setzen Sie die Schaltfläche Alle, um Umbrüche für Titelobjekte auf allen Gliederungsebenen einzurichten.

Benutzerdefinierte Funktionen

Python-Programmierer können eine benutzerdefinierte Funktion angeben, die für jedes ausgewählte Objekt aufgerufen wird. Das erspart das Schreiben von Code, um zu den entsprechenden Elementen zu navigieren. Geben Sie die anzuwendende Funktion wie folgt ein:
Modulname.Funktionsname

Benutzerdefinierte Funktionen können über Parameter verfügen. Um einen oder mehrere Parameter anzugeben, schließen Sie die Parameterliste in Klammern ein und geben die Parameternamen und -werte an. Beispiel:
myfunctions.writeitemtitles(file='filespec')
Dadurch wird die Funktion writeitemtitles im Modul myfunctions aufgerufen. In der Dokumentation finden Sie Informationen dazu, welche Parameter die einzelnen benutzerdefinierten Funktionen zulassen. Parameternamen müssen in Bezug auf die Groß- und Kleinschreibung genau so geschrieben werden, wie dies in der Funktionsdefinition angegeben ist.

Wenn der Parameterwert eine Zeichenfolge ist, schließen Sie ihn in einfache Anführungszeichen ein, z. B.
option='both'

Weitere Funktionen

In diesem Dialogfeld wird Syntax für den Erweiterungsbefehl "SPSSINC MODIFY TABLES" generiert. Führen Sie folgende Syntax aus, um Hilfe für diesen Befehl anzuzeigen:

SPSSINC MODIFY TABLES /HELP.

Die Befehlssyntax bietet folgende Möglichkeiten:
  • Angabe einer Liste von Befehlen für die Auswahl
  • Angabe einer Liste von Tabellenuntertypen
  • Verarbeitung sichtbarer, nicht sichtbarer oder aller Ausgabeelemente
  • Löschen ausgewählter Elemente (anstatt sie nur auszublenden)

Anforderungen

Für dieses Dialogfeld sind das Integrations-Plug-in für Python und der Erweiterungsbefehl "SPSSINC MODIFY TABLES" erforderlich. Bei IBM SPSS Statistics 19 und höher werden das Plug-in und der Erweiterungsbefehl zusammen mit dem Paket "Essentials for Python" installiert. Weitere Informationen finden Sie im Hilfesystem zu IBM SPSS Statistics unter "Core-System > Häufig gestellte Fragen" im Thema "Beziehen von Integrations-Plug-ins". Wenn Sie "SPSSINC MODIFY TABLES" von der SPSS Community heruntergeladen haben, befolgen Sie die Anweisungen in der zugehörigen Readme-Datei.
